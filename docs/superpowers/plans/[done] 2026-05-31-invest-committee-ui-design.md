## 1. 委员会直播 Tab (CommitteeLiveTab.svelte)

### 布局结构

```
┌─────────────────────────────────────────────────────┐
│ [▶ 运行全部 (HOLD + WATCH: 5个标的)]  [状态: 就绪]  │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌─ 💼 Portfolio 持仓摘要 (共享) ────────────────┐  │
│  │  持仓 3只 | 总市值 ¥175,000 | 现金 ¥50,000   │  │
│  │  应急 ¥20,000 | 集中度: 安靠智电 32% ⚠️      │  │
│  └────────────────────────────────────────────────┘  │
│                                                     │
│  ┌─ 安靠智电 (HOLD) ─────────────────────────────┐  │
│  │  ● Macro → ● REGIME → ● Quant R1 → ...       │  │
│  │  [展开查看每个步骤卡片]                         │  │
│  └────────────────────────────────────────────────┘  │
│                                                     │
│  ┌─ 中概互联 (ETF/WATCH) ────────────────────────┐  │
│  │  ● Macro → ● REGIME → ○ Quant R1 → ...       │  │
│  └────────────────────────────────────────────────┘  │
│                                                     │
│  ... 更多标的 ...                                    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**设计要点：** Portfolio 持仓摘要是全局共享数据，对所有标的都一样，因此提到顶部作为独立卡片，不在每个标的内重复展示。

### 关键 UI 元素

**1. 顶部操作栏**
- "运行全部"按钮：主色调绿色，显示 HOLD + WATCH 数量
- 运行中显示进度：`运行中 3/5...` + 取消按钮
- 禁用状态：无持仓时灰色 + tooltip "请先添加持仓"

**2. 标的卡片（每个 HOLD/WATCH 一个）**
- 卡片头部：中文名称 + 资产类型标签（股票/ETF）+ 状态徽章（HOLD 绿/WATCH 琥珀）
- 卡片副标题：代码 `300617.SZ`（灰色小字）
- 流程进度条：7 个步骤圆点（Macro → REGIME → Quant R1 → Risk R1 → Quant R2 → Risk R2 → CIO），用颜色区分状态
  - `●` 绿色 = 完成
  - `◉` 蓝色脉冲 = 进行中
  - `○` 灰色 = 待执行
  - `✕` 红色 = 失败
- 点击卡片展开详细步骤

**3. 展开后的步骤卡片**
每个步骤一个卡片，垂直排列：

```
┌─ Macro 宏观分析 ─────────────────── ✅ 12s ─┐
│ 📊 调用了 get_macro_snapshot, get_history_data │
│                                                │
│ 宏观环境：中性偏多。美联储维持利率...           │
│                                                │
│ 信号：BULLISH | 置信度：72%                     │
└────────────────────────────────────────────────┘

┌─ REGIME 状态计算 ─────────────────── ✅ 0.1s ─┐
│ 波动率：中等 | 均线趋势：上行 | RSI：58        │
│ REGIME：TRENDING_UP                            │
│ 硬性保护规则：无触发                            │
└────────────────────────────────────────────────┘

┌─ Quant 量化分析 R1 ──────────────── ✅ 45s ──┐
│ 🛠️ 调用了 analyze_multi_timeframe              │
│                                                │
│ 技术面：多头排列，MACD金叉...                   │
│ 建议：BUY | 目标价：¥45.00                     │
│ REGIME 硬保护：未触发                           │
└────────────────────────────────────────────────┘

... Risk R1, Quant R2, Risk R2 类似 ...

┌─ CIO 最终裁决 ───────────────────── ✅ 30s ──┐
│                                                │
│ 裁决：BUY | 置信度：78%                        │
│ Sanity Check：4/4 Gate 通过                    │
│ 集中度哨兵：未触发                              │
│                                                │
│ [📋 查看完整报告]                               │
└────────────────────────────────────────────────┘
```

**注意：** Portfolio 持仓摘要已在顶部作为共享卡片展示，不在每个标的的步骤中重复。

**4. 工具调用指示**
- 步骤卡片顶部显示工具调用摘要（如 `🛠️ 调用了 2 个工具`）
- 点击可展开工具详情（输入参数 + 输出结果）
- 与 CommitteeToolsTab 联动

### 交互流程

1. 用户点击"运行全部"
2. 后端并行/串行处理每个标的
3. 前端通过 SSE 事件流接收每个标的的每步骤进度
4. 步骤卡片逐个点亮，进行中的步骤有脉冲动画
5. 全部完成后，CIO 裁决卡片显示最终结果
6. 结果自动归档（无需用户操作），每天每个标的仅保留最新一次裁决

### 与现有代码的集成点

- `invest-committee-store.svelte.ts` → `runCommittee()` 改为运行全部 HOLD+WATCH
- `run_committee_stream` SSE 事件 → 新增 `step_progress` 事件类型
- `events.rs` → 更新 step_index 映射（新增 REGIME 步骤，7 步流程）
- `orchestrator.rs` → REGIME 注入后发送对应事件；Portfolio 作为共享上下文注入（非流程步骤）

---

## 数据保留策略

- **自动归档**：委员会运行完成后自动归档，无需用户操作
- **每天最新覆盖**：每天每个标的仅保留最新一次裁决数据，新运行覆盖旧数据
- **回放展示全流程**：回放时展示全部 7 步完整数据（Macro → REGIME → Quant R1 → Risk R1 → Quant R2 → Risk R2 → CIO）
- **模拟执行不归档**：Dry Run 结果仅展示不保存

## 2. 委员会回放 Tab (CommitteeReplayTab.svelte)

### 布局结构

```
┌─────────────────────────────────────────────────────┐
│ [真实回放] [模拟执行]     标的: [安靠智电 ▼]        │
├─────────────────────────────────────────────────────┤
│                                                     │
│  模式 A: 真实回放                                    │
│  ┌─ 最近裁决: 2026-05-30 14:30 ──────────────────┐  │
│  │ 裁决: BUY | 置信度: 78% | 归档时间: 3天前     │  │
│  └────────────────────────────────────────────────┘  │
│                                                     │
│  [Portfolio 共享] [Macro] [REGIME] [Quant R1] ... 卡片 │
│  (静态展示，无动画，从数据库读取)                     │
│                                                     │
│  ── 或 ──                                           │
│                                                     │
│  模式 B: 模拟执行                                    │
│  ┌─ Round 次数: ○1 ●2 ○4 ○6 ○8 ────────────────┐  │
│  │ 1=最小 2=交叉审查 4=推荐 6=极限 8=实验        │  │
│  └────────────────────────────────────────────────┘  │
│                                                     │
│  [▶ 开始模拟]  (结果带"模拟"水印，不进归档)          │
│                                                     │
│  [Macro] [REGIME] ... 卡片 (带流程动画)              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 关键 UI 元素

**1. 模式切换**
- 顶部 Tab 切换：`真实回放` | `模拟执行`
- 切换时清空下方内容区

**2. 标的选择器**
- 下拉框，显示中文名称（从 holdings 表获取）
- 只显示 HOLD + WATCH 标的
- 每个选项：`安靠智电 (300617.SZ) | HOLD`

**3. 真实回放模式**
- 顶部信息卡：最近裁决日期、裁决结果、置信度
- 步骤卡片：与直播相同的卡片布局，但从数据库读取历史数据
- 无动画，直接显示结果
- 工具调用历史从归档数据中读取
- 底部操作：`[📋 导出报告]`

**4. 模拟执行模式**
- Round 选择器：单选按钮组，每个选项有简短说明
  - `1` = 最小（单轮）
  - `2` = Cross-Examine（交叉审查）
  - `4` = 推荐（默认选中）
  - `6` = 极限
  - `8` = 实验
- "开始模拟"按钮
- 执行过程与直播相同，但：
  - 结果卡片右上角显示 `[模拟]` 水印标签（琥珀色）
  - 不显示"归档"按钮
  - 底部显示 `⚠️ 模拟结果仅供参考，不进入归档`

**5. 空状态**
- 无历史裁决时：`暂无裁决记录，请先运行委员会`
- 无持仓时：`请先添加持仓标的`

### 交互流程

**真实回放：**
1. 用户选择标的
2. 前端从 DB 查询该标的最近一次裁决的全部步骤数据
3. 步骤卡片静态渲染，每个卡片可展开查看详细输出

**模拟执行：**
1. 用户选择标的 + Round 次数
2. 点击"开始模拟"
3. 前端调用 `run_committee_stream`（dry_run=true 参数）
4. 步骤卡片带动画逐个完成
5. 结果显示但不归档

### 与现有代码的集成点

- `CommitteeReplayTab.svelte` → 完全重写
- `commands/invest.rs` → 新增 `dry_run` 参数到 `run_committee_stream`
- `orchestrator.rs` → dry_run 模式跳过归档步骤
- `storage/invest/committees.rs` → 查询最近裁决的完整步骤数据

```

│                                                     │

│ 持仓数: 3只 (HOLD)  观望数: 2只 (WATCH)             │
│                                                     │
├─────────────────────────────────────────────────────┤
│                                                     │
│ 名称         类型  状态  数量  成本  现价  盈亏%     │
│ ─────────────────────────────────────────────────   │
│ 安靠智电     股票  HOLD  500   38.2  42.5  +11.3%  │
```

### 关键 UI 元素

**1. 汇总卡片**
- 总市值、总盈亏（绿色正/红色负）
- 持仓数 (HOLD) + 观望数 (WATCH)
- 现金、应急储备、可用资金

**2. 合并表格**
- 状态列：`HOLD` 绿色标签 / `WATCH` 琥珀色标签
- 类型列：`股票` / `ETF` 标签
- HOLD 行在上，WATCH 行在下，中间有分隔线
- WATCH 行数量/成本/盈亏列显示 `--`（未持有）
- 盈亏列：正数绿色，负数红色
- 排序：HOLD 优先，然后按盈亏降序

**3. 操作列**
- HOLD 行：`[卖出]` `[转为观望]` `[运行委员会]`
- WATCH 行：`[买入]` `[移除]` `[运行委员会]`
- 所有操作按钮使用图标 + tooltip

**4. 添加标的对话框**
- 资产类型选择：股票 / ETF
- 代码输入（带自动补全）
- 名称自动填充
- 初始状态：HOLD / WATCH

### 交互流程

1. 页面加载时自动刷新价格
2. 点击行展开详情（日线图、最近裁决、交易记录）
3. 点击"运行委员会"触发单标的委员会运行
4. 买入/卖出弹出 TradeDialog

### 与现有代码的集成点

- `HoldingsTable.svelte` → 合并 HOLD/WATCH，添加类型列
- `invest-store.svelte.ts` → `loadHoldings()` + `loadWatchlist()` 合并为 `loadAllAssets()`
- `TradeDialog.svelte` → 支持 ETF 买入
- `portfolio.rs` → `list_assets()` 查询合并

4. 供应商配置修改后自动保存
5. "恢复默认"按钮弹出确认对话框

### 中文化策略（重要）

**UI 布局**：参考 Demo 的样式（R1/R2 Tab、折叠面板、工具标签）

**实际内容**：保留现有专业指标信息，不做简化。Demo 中的中文描述仅作布局示意。

**LLM 输出中文化**：在 Prompt 模板中要求 LLM 使用中文输出关键字段，例如：
- `Signal: BULLISH` → `信号: BULLISH`
- `Confidence: 78%` → `置信度: 78%`
- `Risk Level: MEDIUM` → `风险等级: 中等`
- `Verdict: BUY` → `裁决: BUY`

**前端展示**：LLM 返回结果的标签用中文（信号、置信度、风险等级、裁决等），数值保持原样。

### 与现有代码的集成点

- `CommitteeRolesTab.svelte` → UI 布局改造（R1/R2 Tab），保留现有专业内容

### 状态颜色

| 状态 | 颜色 | 用途 |
|------|------|------|
| 完成 | `#22C55E` | 步骤完成、HOLD 状态、正盈亏 |
| 进行中 | `#3B82F6` | 当前执行步骤（脉冲动画） |
| 待执行 | `#64748B` | 未开始的步骤 |
| 失败 | `#EF4444` | 步骤失败、负盈亏 |
| 观望 | `#F59E0B` | WATCH 状态、模拟水印 |
| 信息 | `#8B5CF6` | REGIME 数据卡片 |
| 持仓 | `#06b6d4` | Portfolio 共享数据卡片 |

### 动画规范

| 动画 | 时长 | 缓动 |
|------|------|------|
| 卡片展开/折叠 | 200ms | ease-out |
| 步骤状态切换 | 150ms | ease-in-out |
| 进行中脉冲 | 1500ms | ease-in-out infinite |
| 模拟水印淡入 | 300ms | ease-out |
| Toast 通知 | 3000ms auto-dismiss | ease-out |

### 无障碍规范

- 所有交互元素 ≥ 44px 触摸目标
- 颜色对比度 ≥ 4.5:1
- 键盘导航支持 Tab 顺序
- `aria-label` 用于图标按钮
- `prefers-reduced-motion` 尊重
