# 每日盈记（Fortune Journal）设计文档

- 日期：2026-07-10
- 状态：**已实现**（v5.6.10+）
- 定位：openInvest 下的偏娱乐功能。用户按天干地支记录每日股市收益率，累积后发掘干支与收益的规律。

## 1. 目标与范围

按公历日的**日干支**（甲子…癸亥共 60 种）记录每日收益率，展示：
- 今日/明日的干支综合评分（吉凶）
- 干支排行/风险（天干、地支各 Top3）
- 数据统计（KPI、Top3、月度）
- 可选的 AI 文字解读

### 已确认决策

| 项 | 结论 |
|---|---|
| 收益率来源 | 纯手动输入（不自动带出 pnl_snapshots） |
| 干支粒度 | 日干支（每天一个） |
| 存储 | 复用 `invest.db`，新增两张表（不建独立 fortune.db） |
| AI 解读 | 复用委员会 Claude CLI executor，**手动按钮**触发，异步 |
| 定时任务 | 暂不接 scheduler |
| i18n | **只做中文**；但 en.json 放同名中文值作 fallback（满足 `i18n:check`） |
| 历史评分语义 | **A 方案：point-in-time 冻结**（见 §3） |
| 录入 | 弹窗默认录当日，可切批量模式；批量左侧固定工作日日期行、右逐行填收益 |
| 批量日期范围 | **每页一个自然月的工作日，按月翻页**；默认当前月且只到今天，不可翻未来；已录入日期预填旧值可覆盖(upsert)，视觉标记"已录/空" |
| 排行/风险 | 天干、地支各 **Top3** |
| 综合分曲线 | **线性放大 + 一致度回拉**（定稿见 §9.2）：综合分 = G(天干层分, 地支层分)，天干地支都好才拉高、背离则抵消 |
| 小样本 | **层分不做收缩**（饱和曲线本身收极端值）；冷启动锐度靠展示门控处理（定稿见 §9.1） |
| 休市日 | 复用现有 `trade_calendar` 表(`is_open`)判定，缺数据回退周末 heuristic；仅日历标灰，录入界面所有工作日均可填 |
| AI 解读 | 独立 1 个 CLI 信号量(不与委员会抢)；生成中按钮禁用+解读卡 skeleton，不支持取消；同一天可多次生成(追加)，`get_reading` 返回最新一条 |

### 非目标（YAGNI）

- 不做英文翻译
- 不接定时提醒/自动占位
- 不做时辰/月/年干支，只做日干支
- 不自动同步组合盯市收益

## 2. 评分算法（Phase 0，核心已完成并测试）

只有**两样量有独立意义**：层分（每个天干/地支单独多好，排行榜展示、由用户排行数据标定）与综合分（当天两个层分的一个函数）。中间不引入"融合分"这类命名概念——合并只是综合分函数内部的一个算术步骤。数据链：

> 原始(平均收益, 胜率) → **天干层分 S、地支层分 B** → **综合分 = G(S, B)**

> **算法已定稿并落地**（2026-07-11）。曲线的最终公式、常数、拟合/验证数据见 **§9**（本节仅保留概念框架与实现须知）。`stats.rs` 已按 §9 更新完毕、测试全绿。

### 2.1 层分（统一饱和曲线，定稿见 §9.1）

```
平均收益% = 该干支所有样本日收益率的算术平均
胜率%     = 收益 > 0 的天数 / 样本天数 × 100      # 口径已钉死：仅正收益算赢，平盘(=0)不算赢
层分      = 见 §9.1（raw=10.6·avg+1.08·(win−50)，饱和到 0/100，无收缩）
```

- 50 = 中性锚点（零收益、50% 胜率）。
- **胜率口径**：`收益 > 0` 才计入赢，平盘（=0）不算赢。后端聚合须严格按此，否则会与用户权威排行数偏离（demo 曾因口径不一致把酉胜率算成 57% vs 权威 50%，导致综合分跨错吉凶带）。
- **无小样本收缩**：饱和曲线本身把极端值收在 0/100 间；冷启动噪声（n≤2 的极端排名）由**前端展示门控**处理（n<阈值标「数据积累中」），不改分数。

### 2.2 综合分（线性放大 + 一致度回拉，定稿见 §9.2）

```
s = 天干层分 − 50,  b = 地支层分 − 50
综合分 = 见 §9.2（56.9 + 1.191·s + 0.887·b − 0.137·sign(s+b)·|s·b|/10，clamp[0,100]）
吉凶带 = ≥75大吉 / 60吉 / 45平 / 30凶 / <30大凶
```

**为什么综合分需要"放大"而非简单平均**：基准点里两个层分都略高于 50，用户目标分却**高于两个层分**——任何加权平均都够不到。故必须把"两者都略好"放大成"这天相当好"（证据叠加）。反之两层背离（一高一低）则相互抵消、停在中性（如基准点 丁亥 层分 24.8/97.3 却只 53）。这正是用户打分的直觉，公式编码的就是这个规律。

- 消融实验（新旧层分值两轮）证明「干支组合」层对综合分拟合零贡献，故**不进综合分**（仍统计展示 avg/胜率/次数）。schema 也不含组合分。
- 常数集中在 `stats.rs` 顶部命名常量，可用真实数据回调。
- 已落地：`invest/fortune/calendar.rs`（干支，历法锚点测试）、`invest/fortune/stats.rs`（层分/综合分，§9 定稿版，7 测试全绿）。
- 本机跑 Rust 测试走 `node scripts/rust-test.mjs`（嵌 manifest 解决 0xc0000139，见 CLAUDE.md §11）。

## 3. 架构与数据流

```
用户录入收益率 ──► fortune_daily_returns (invest.db)
                        │
                        ▼
        aggregate.rs：按日期从早到晚单趟扫描，边扫边维护
        每个天干/地支/组合的累加器(收益和/天数/盈利天数)
                        │
        ┌───────────────┼────────────────┐
        ▼               ▼                ▼
   层分(天干/地支)   综合分曲线      干支排行/风险表
        │               │                │
        └──── 分析页 ────┘         干支总览 / 数据总览
                │
       手动「生成解读」──► 委员会 CLI ──► fortune_ai_readings ──► 右侧解读卡
```

### Point-in-time 语义（A 方案）

某日 D 的综合分 = **只用 D 之前的记录**聚合出的天干/地支统计。这与用户给的「预测分 / 盘后分」基准逻辑一致：

- **今日卡（盘前/盘中）**：用今天之前的数据 → 预测分
- **今日卡（盘后）**：今天收益录入后纳入统计 → 盘后分
- **明日卡**：用「今天及之前」数据预测 → 预测分
- **日历历史格**：该日的预测分（**一旦算出永不变**，不受后续新数据影响）
- **日历未来格**：用当前全量数据预测

**单趟累加器技巧**：扫到某天 D 时，先给 D 拍快照（此刻累加器只含 D 之前数据）= D 的预测分；再把 D 的收益加进累加器 = D 的盘后分。一趟 O(N) 同时得到每天的预测分与盘后分，无需嵌套重算。扫完后的最终累加器（含全部记录）用于排行/风险表、明日预测、未来预告。

> ⚠️ **后端必做**：point-in-time 是 `aggregate.rs` 的核心，UI demo 未实现它（demo 只用全量数据算单一分数）。验证过：喂入用户真实盘前统计得预测分 ≈78、纳入当日 −2.63% 后盘后分 ≈68，落差 −10 与用户基准吻合——**只要后端做对 point-in-time + 胜率口径(收益>0)，就能复现该落差**。demo 的偏差来自缺这两项，非评分公式问题。

## 4. 数据模型

```sql
CREATE TABLE fortune_daily_returns (
    date        TEXT PRIMARY KEY,   -- 'YYYY-MM-DD'
    return_pct  REAL NOT NULL,      -- 当日收益率(%)
    note        TEXT DEFAULT '',
    created_at  TEXT NOT NULL,
    updated_at  TEXT NOT NULL
);
CREATE TABLE fortune_ai_readings (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    date        TEXT NOT NULL,      -- 解读对应哪一天
    content     TEXT NOT NULL,
    created_at  TEXT NOT NULL
);
```

- 周**从周日开始**，顶部 `‹ 年月 ›` 按月翻动。

### 干支预告（6 路）

扫描未来工作日，给出 6 个极值各自**所在交易日**（日期 + 周几 + 吉凶带分）：下个最强天干、最弱天干、最强地支、最弱地支、最强组合、最弱组合。强用红左边、弱用绿左边。

### 批量录入（按月翻页）

左侧固定工作日日期行、右侧逐行填收益；**每页展示一个自然月的工作日（周一~周五，排除周末）**，`‹ 上一月 / 下一月 ›` 翻页，顶部显示当前月份（如「2026年7月」）。

- **默认页 = 当前月**；不能翻到未来月份（下一月按钮到当前月即禁用）。
- **当前月只显示到今天（含）为止的工作日**，今天之后的工作日不列出（避免录入未来数据）。历史月份显示该月全部工作日。
- 已录入行红点标记 + 预填旧值可覆盖（upsert）；空行留白。

### 数据总览指标

- **KPI 字段**：总记录天数、盈利天数、胜率%、累计收益%、平均日收益%。
- **月度统计**：月均收益柱状图（每月一根，**红涨绿跌** `--up`/`--down`）。

### 空状态（首次无数据）

- 分析 tab：今日/明日卡显示引导文案「录入第一笔收益后即可查看干支评分」；日历格无评分标签。
- 干支总览 / 数据总览：排行/KPI 显示「数据不足」占位。

## 9. 算法标定进展（与用户逐步捋）

> 方向变更：用户对参考 demo 的评分手感（0~100 铺满、决断力强、小样本也打分）更满意，据此重定算法。旧的「线性层分 + n/5 收缩 + 一维 m 合并」被下述**统一饱和曲线**取代。demo 无源码，全靠喂点反推 + 最小二乘验证。

### 9.1 层分：统一饱和曲线（✅ 已定稿，三窗口验证通过）

关键发现：用户先后给的多份标定（大 dataset 层分排行 avg≈0；冷启动 demo avg≈±5）**并不矛盾，是同一条饱和曲线的两段**——近 0 处陡、两端饱和压向 0/100。之前误以为要两套线性常数（15.82 vs 7.85），实为量了同一曲线的不同区段。

```
raw   = 10.6·平均收益% + 1.08·(胜率%−50)
层分  = 50 ± 61·(1 − e^(−0.020·|raw|))     # raw≥0 取+ / <0 取−；天然铺满 0~100
胜率% = 收益>0 天数 / 样本天数 × 100        # 口径已钉死（=0 不算赢）
```

- **无需小样本收缩**：饱和曲线本身把极端值收在 0/100 之间；收缩正是把 demo 锐度摁没的元凶，去掉后反而对齐用户满意的 demo。
- **拟合**：全量181天 12 点 + 6月 12 点 = 24 个层分点，RMSE **2.16**，最大偏差 5.2，无需剔任何离群点。
- **留出验证**：5+6+7月窗口（47 天，曲线拟合时未见）12 个排行点 RMSE **1.88**，全部落在 ±3 内。三个数据窗口同一条曲线通吃，泛化良好。
- **踩过的坑**：曾因漏读一条空值行（6/19 无数据被当成"甲少一天"），把甲误标 85.48 → 造出"甲 avg 高分反低"的假反常，一度引入"波动率惩罚"歧路。修正数据后甲=100 完全单调，假设作废。教训：拟合前先核对每个标定点的原始统计。
- **top 区分辨率**：巳(2.02%)=97.5 vs 亥(1.985%)=97.4，饱和区压缩使高分榜排序差距极小；娱乐场景可接受，若需精确榜序需另想办法。
- **冷启动代价**：无收缩 ⇒ 壬(1天+2.59%)→99 等噪声排名。属产品取向（决断力 > 精确），若需缓和用**展示门控**（n<阈值标「数据积累中」，不改分数），不动曲线。

### 9.2 综合分：结构已明，待用新层分值重拟合（⏳ 进行中）

**核心机制已由数据坐实——天干地支「越一致、放大越强」，单条强层分扛不起综合分**：

| 点 | 天干层 | 地支层 | 关系 | 目标综合 | 说明 |
|---|---|---|---|---|---|
| 乙亥预测 | 55 | 59 | 都中高·一致 | 78 | 一致向上 → 强放大 |
| 丁亥(5+6+7月) | 24.8 | 97.3 | 一低一高·背离 | 53 | 高层被抵消 → 停中性 |
| 甲戌预测 | 38 | 64 | 背离 | 43 | 抵消 → 偏低 |
| 丁丑盘后 | 25.5 | 29.2 | 都低·一致 | 9 | 一致向下 → 强负放大 |

- **交互方向**：错误用 `s·b`（对"都正""都负"同号，负向放大反被顶回）；正确用 `sign(s+b)·|s·b|`，顺一致方向放大。
- **组合参数近乎无用**：消融显示加组合项 RMSE 仅 7.77→7.37，系数也小 → **天干+地支基本够用，组合可砍以简化**（待新层分值复核后定）。
- **冷启动 vs 成熟点冲突**：6月点（如丁亥6月 亥层97.5 但综合仅53）与成熟点不服同一放大规律，混拟合会两头妥协；剔冷启动后成熟 17 点 RMSE 可降到 ~5.3。是否对冷启动单独处理待定。

**定稿公式（成熟 19 点分权拟合，RMSE 5.07 / 最大偏差 9.9）：**

```
s = 天干层分 − 50,  b = 地支层分 − 50
综合 = 56.9 + 1.191·s + 0.887·b − 0.137·sign(s+b)·|s·b|/10     # clamp[0,100]
```

- 天干权重 1.191 略重于地支 0.887；一致度项 `sign(s+b)·|s·b|/10` 负系数在极端一致时轻微回拉，防冲爆 0/100。
- 中性锚 = 56.9（层分均 50 时；demo 略偏乐观，中性日落"平"偏上）。
- **组合参数确认剔除**：新旧层分值两轮消融均显示零贡献，schema 与算法都不含组合分。
