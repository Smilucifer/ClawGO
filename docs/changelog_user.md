# 更新日志

面向用户的版本说明。技术实现细节请查阅 `docs/changelog.md`。

## v5.5.11 (2026-06-24)

### 涨跌家数数据修复

- 修复「上涨家数」和「下跌家数」两项市场宽度指标因上游数据接口格式变化而无法正常获取的问题，委员会宏观快照中的涨跌家数恢复正确显示。

## v5.5.10 (2026-06-24)

### 定时任务修复（续）

- 修复 v5.5.9 未完全解决的定时任务不触发问题。上一版虽然补全了磁盘结构，但任务的下次执行时间在程序启动时仅存在于内存、从未写入磁盘，导致每次重启后所有 cron 任务（盈亏快照、事件扫描、每日报告、宏观刷新等）再次陷入"永远不满足触发条件"的死循环。现在程序启动时会立即把计算好的下次执行时间写入磁盘，确保重启后任务按计划正常触发。

## v5.5.9 (2026-06-24)

### 定时任务修复

- 修复自 6 月 10 日以来所有定时任务（盈亏快照、事件扫描、每日报告等）无法自动触发的问题。原因是程序重启后任务的下次执行时间被错误重置为未来某个时刻，导致永远不满足触发条件。现在下次执行时间会正确持久化，重启后恢复原计划。

### 宏观指标升级

- 宏观指标中的「沪深 300」已替换为「上证指数」（上证综指），数据源同步切换。委员会分析中的市场阶段判定规则、宏观快照等均已更新。
- 新增两项市场宽度指标：**上涨家数**和**下跌家数**，反映全市场涨跌股票数量，已纳入委员会宏观快照。

## v5.5.8 (2026-06-23)

### 委员会最终裁决卡片排版优化

- 投资委员会的最终裁决正文现在支持 Markdown 渲染（标题、加粗、列表、表格等），不再是一整段挤在一起的纯文本，阅读体验更清晰。

## v5.5.7 (2026-06-22)

### 持仓清仓保护期

- 持仓股全部卖出（清仓）后，当日依然显示为「持仓」状态，并标注「已清仓」标记。当日盈亏、当日盈亏比等指标照常计算，不会因清仓归零。
- 次日凌晨 5:05 自动将已清仓的持仓转换为「关注」。如果在清仓当日又重新买入（做T），保护期自动取消，不会误转换。
- 手动将持仓股数编辑为 0 同样适用上述保护期逻辑。
- 修复：之前很久以前清仓的股票会在 dashboard 持续显示「已清仓」标记，原因是交易重放时清仓日期被错误刷新为当天。现已修复，历史清仓条目会在次日正确转换为「关注」。
- 修复：删除观望列表中的条目后，下次打开或定时任务运行时条目会重新出现。现已修复，删除操作会持久生效。

## v5.5.5 (2026-06-20)

### 投资委员会

- 修复委员会直播卡片频繁误报"部分字段未识别"、并据此把最终评级无故降级为"观望(HOLD)"的问题。此前风控、首席投资官等角色在分析里用了加粗、标题等排版,系统就读不出关键字段,即便分析本身完整也会被判定为异常而降级。现在能正确识别这类排版,评级回归各角色的真实结论(只有在分析节点真正不可用时才会降级)。
- 风控 R1 和最终裁决卡片现在也像宏观、量化卡片一样,把集中度、可用子弹、盈亏、最大回撤、止损、执行计划等字段分段清晰展示,不再是一整段挤在一起的文字。

### 其他

- "用量"页面余额卡片排版微调:DeepSeek 与 PackyAPI 各占半行(两者合计宽度与小米卡片对齐),小米卡片独占整行,排布更整齐。

## v5.5.4 (2026-06-20)

### 投资委员会分析模式

- 委员会直播卡片新增「研究 / 实盘」分析模式一键切换：
  - **实盘模式**：站在你自己账户的视角分析，会考虑你的现金、持仓集中度与真实成本，裁决对应「该怎么加减仓」
  - **研究模式**：抛开你的持仓与现金，纯粹评估这只标的本身值不值得买，裁决对应「值得买 / 观望 / 规避」
- 默认按是否持仓给出初始模式（关注的票默认「研究」、持仓的票默认「实盘」），任何标的都能手动改，切换后会显示「手动设置」标记并在重启后保留

### 记忆系统

- 修复"记忆"页面在左侧选中某个项目后，无法正确列出该项目下记忆文件的问题
- 普通聊天现在也会自动从对话中提炼记忆（此前仅群聊生效），并新增"手动添加记忆"入口

### 投资委员会直播

- 重新设计直播卡片：市场状态（REGIME）上移到卡片头部一眼可见，进度条收窄不再占满整行
- 卡片展开后，各角色分析改为结构化卡片展示（信号、置信度、买点/估值/催化等字段一目了然），比原先的纯文本更清晰；并在不同屏幕宽度下自适应排版

### 功能精简与系统

- 移除业务上已无用的"委员会-Tool调用"和"系统-市场Regime"两个页签
- 新增"重启 Python 运行时"按钮，Python 卡死时无需重启整个应用
- 系统页新增"数据清理"功能：扫描并在确认后清理无用的历史数据（空转的投资洞察记录、只写不读的旧表、遗留目录）
- 数据源检测补全：新增腾讯实时行情/K线、Tushare 新闻、AkShare 国债与涨跌停、Yahoo 历史、Python 运行时等检测项，问题定位更直观

### 用量与余额

- `/usage` 页面在 DeepSeek 右侧新增 PackyAPI 余额卡片，可查看额度余量
- 小米与 PackyAPI 的登录凭据现在会在每次刷新时自动续期，显著减少"余额查询失效、需重新手动粘贴 cookie"的情况
