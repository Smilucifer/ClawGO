# 更新日志

面向用户的版本说明。技术实现细节请查阅 `docs/changelog.md`。

## v5.6.8 (2026-07-08)

### 新增

- **盘前观察报告：** `/invest` 的「系统」下新增「盘前观察」子页,面向下一交易日生成一份图文报告。报告分四段:①外部/舆论新闻先验(多源抓取聚合成板块 + 评价标签)②资金与宏观双重确认(宏观快照 + 板块资金流入榜 + 拥挤度雷达)③明日主线排序(板块聚合)④S/A/B/C 观察池分级。硬数据由确定性打分给出、AI 只负责解读话术,评级不受 AI 影响。支持一键立即生成、导出 PNG / PDF,可在设置面板自定义四因子权重与 SABC 阈值。工作日盘前 9:00 自动生成。
- **多源舆情采集:** 新增同花顺 / 新浪 / 财联社 / 东财股吧 / 雪球五源舆情抓取,统一去重入库并经 AI 归一化(提炼摘要、判断情绪立场、映射所属行业)。雪球通过隐身浏览器过反爬,登录 Cookie 加密存储;抓不到时自动降级,不影响报告产出。委员会的个股催化分析也改为从这份舆情池 + 事件池读取(个股 + 行业两路),数据更全。

### 调整

- **全局红涨绿跌:** 投资模块的涨跌配色统一为「红涨绿跌」(涨/正 = 红,跌/负 = 绿),与盘前报告及 A 股用户习惯一致。成功/错误等状态色不受影响。
- **宏观判断改为全局单卡:** 委员会的宏观解读从每只标的重复展示,改为一张全局宏观卡片 + 每标的仅显示敏感度小条,减少重复、更清晰。

## v5.6.7 (2026-07-08)

### 移除

- **移除「记忆管理」页面：** 侧边栏与首页的「记忆管理」入口已下线。该页面用于手动查看和增删自动记忆，实际帮助有限。基于文件的 `/memory`（`MEMORY.md` 编辑器）不受影响，照常可用。

### 调整

- **自动记忆默认关闭：** 从聊天中自动提取记忆、后台整理记忆（做梦）的功能现在默认关闭，不再自动运行。相关能力仍保留，如需重新开启可在配置中调整。

## v5.6.6 (2026-06-29)

### 修复

- **宏观快照 20 日波动率不再显示 N/A：** 之前由于对数收益率公式错误，上证指数 20 日波动率（`sh_composite_vol20`）在价格下跌时计算结果为 NaN，导致宏观快照始终显示 N/A。现已修正公式，波动率可正常展示。
- **GitHub Release 安装包补全 Python 依赖：** 从 GitHub 下载的安装包此前缺少 `akshare` 库，导致涨停/跌停家数、上涨/下跌家数、国债收益率（fallback）等 4 个宏观指标在其他电脑上获取失败。现已补全，安装包体积与本地构建一致。
- **清仓后重新买入的标的不再丢失：** 之前如果一只股票从关注列表删除后重新买入，清仓时该持仓会直接消失，不会按预期保留到次日转为关注。现已修复，清仓后持仓会正常保留并在次日自动转为关注状态。

## v5.6.4 (2026-06-27)

一次全面的代码审查修复，覆盖隐私、安全、投资委员会准确性、记忆系统与稳定性。

### 隐私与安全

- **私聊不再被公开：** 群聊中用 `/dm @某人 消息` 发的私信此前会被错误地当成公开广播发给所有成员，现已修复，私信只发给指定对象。
- **私聊不再进入全局记忆：** 私聊内容此前可能被自动提取并写入共享的全局记忆，现已完全跳过，私聊文本不再外发或留存到记忆。
- **凭证更安全：** 修复了启动时在极端情况下可能清空已保存凭证（API Key、Token 等）的隐患；凭证不再写入调试日志；临时配置文件不再残留累积；剪贴板粘贴不再能读取 `.env` 等敏感文件。

### 投资委员会

- **交易记录日期统一：** 交易记录里部分条目（如卖出）日期显示为 `2026/6/26` 斜杠格式，与其他 `2026-06-24` 不一致。现统一为 `YYYY-MM-DD`，且历史数据一并修正。
- **裁决方向更准确：** 修复了委员会裁决中「不要买入」「暂不买入」等被错误识别成「买入」的方向反转问题；置信度、信号强度的数值口径也已校正，不再出现 `7500%` 这类异常显示。
- **宏观信号识别更全：** 「看多」「看空」「避险」等更多表述现能被正确识别，不再被一律归为「中性」而漏掉真实的风险信号。
- **取消分析立即生效：** 取消正在运行的委员会分析时，现在会立即中断对应的子进程并释放资源，不再需要等待超时。
- **数据更可靠：** 修复了北向资金为 0（平盘）被误判为无效数据、周末把最新交易日数据误判为「过期」等问题。

### 记忆系统

- **记忆提取可独立开关 + 质量筛选：** 记忆管理页新增独立的「记忆提取」开关（不再依赖向量嵌入开关）、置信度阈值滑块和保存配置按钮。低置信度、过短、空泛的内容会在入库前被过滤，减少无价值记忆。

### 稳定性

- **进入聊天页崩溃修复：** 修复 v5.6.3 引入的、官方 Claude 订阅用户进入聊天/私聊页时整页崩溃无法进入的问题。
- **会话状态更可靠：** 会话恢复所需的标识、终态状态写入失败时会重试，减少重启后无法恢复会话或状态显示不一致的情况。
- **群聊 @提及更宽容：** `@某人,` `@某人。` 这类带标点的提及现能正确识别成员，不再因尾随标点找不到对象。
- **历史记录更完整：** 单条损坏的会话记录不再导致整条会话从历史列表中消失。
- **多字节文字不再乱码：** 修复长输出中中文等多字节字符在特定边界被替换成乱码符号（�）的问题。
- **Windows 修复：** 修复 Windows 上项目级 / 用户级 agent 的增删改查整体失效的问题。

## v5.6.0 (2026-06-25)

### 数据源编排层 + miniQMT 接入

- **数据源智能降级：** openInvest 的取数系统进行了架构升级，现在当首选数据源返回空值或异常时，会自动降级到备用数据源，无需手动干预。默认优先级为 tushare → AkShare（资金/宏观类）或腾讯（行情类）。
- **miniQMT 行情源支持：** 新增可选的 miniQMT 本地行情数据源。如果你本机运行了已登录券商账户的 miniQMT 客户端，可在「设置 → 投资」中开启「启用 miniQMT 作为行情优先数据源」，K线和实时报价将优先从本地 miniQMT 获取（无限频、本地缓存），取不到时自动降级到在线数据源。未安装 miniQMT 的用户无需任何操作。
- **数据源透明记录：** 宏观指标缓存现在会记录每项数据的真实来源（tushare / akshare / tencent / miniqmt），便于排查数据问题。

### 北向资金修复

- 修复「北向资金净流入」长期显示为 0 的问题。原因是上游接口的一个已废弃字段被错误引用，现已切换到正确字段并修正单位换算（亿元）。

### 当日盈亏计算升级

- **盯市口径重写：** 当日盈亏改用现金流调整盯市法，正确覆盖所有场景：纯持有、当日新买、加仓、减仓、清仓。计算公式直观反映"今天开盘到收盘你实际赚/亏了多少"。
- **手续费方案配置：** 新增可配置的手续费方案（佣金费率、最低佣金、印花税率、过户费率），佣金自动计入当日盈亏的现金视角，使盈亏数字更贴近真实账户。在交易弹窗中可选择手续费方案。

## v5.5.10 (2026-06-24)

### 定时任务修复（续）

- 修复 v5.5.9 未完全解决的定时任务不触发问题。上一版虽然补全了磁盘结构，但任务的下次执行时间在程序启动时仅存在于内存、从未写入磁盘，导致每次重启后所有 cron 任务（盈亏快照、事件扫描、每日报告、宏观刷新等）再次陷入"永远不满足触发条件"的死循环。现在程序启动时会立即把计算好的下次执行时间写入磁盘，确保重启后任务按计划正常触发。

## v5.5.9 (2026-06-24)

### 定时任务修复

- 修复自 6 月 10 日以来所有定时任务（盈亏快照、事件扫描、每日报告等）无法自动触发的问题。原因是程序重启后任务的下次执行时间被错误重置为未来某个时刻，导致永远不满足触发条件。现在下次执行时间会正确持久化，重启后恢复原计划。

### 宏观指标升级

- 宏观指标中的「沪深 300」已替换为「上证指数」（上证综指），数据源同步切换。委员会分析中的市场阶段判定规则、宏观快照等均已更新。
- 新增两项市场宽度指标：**上涨家数**和**下跌家数**，反映全市场涨跌股票数量，已纳入委员会宏观快照。

## v5.5.7 (2026-06-22)

- 修复：之前很久以前清仓的股票会在 dashboard 持续显示「已清仓」标记，原因是交易重放时清仓日期被错误刷新为当天。现已修复，历史清仓条目会在次日正确转换为「关注」。
- 修复：删除观望列表中的条目后，下次打开或定时任务运行时条目会重新出现。现已修复，删除操作会持久生效。
